- 관리자 모드의 적 공격 정지, 무적, 구역 봉쇄와 보스 관문 통과 기능은 그대로 유지

## v1.3.2 추락 처치·관리자 모드

- 일반 적과 보스가 낭떠러지 아래로 떨어지면 생성 위치로 복귀하지 않고 즉시 처치 처리
- 시작 화면에서 숫자 키 `1 → 2 → 1 → 4`를 순서대로 입력하면 관리자 모드 활성화
- 관리자 모드에서는 적과 보스가 이동을 멈추고 공격·접촉 피해를 주지 않으며 플레이어는 피해를 받지 않음
- 구역 봉쇄벽과 보스 관문을 모두 통과할 수 있고, HUD와 시작 화면에 관리자 모드 활성 상태 표시
- 관리자 모드는 페이지를 새로 열기 전까지 새 작전과 이어하기에 적용

## v1.3.1 인게임 관문 컷신·잔영 대화

- 기존의 전면 암전·추상 배경 컷신을 없애고, 실제 플레이 맵과 캐릭터를 그대로 보여 주는 인게임 연출로 교체
- 보스 처치 뒤 다음 스테이지 관문을 지나면 카메라가 플레이어와 새 구역을 함께 비추고, `GATE PASSED` 스테이지 리본과 대화 장면이 이어짐
- 현장 인물은 캐릭터 머리 위 흰색 픽셀 말풍선, 원격 통신과 기록 음성은 반투명 통신 채널 패널·인물 초상으로 구분
- 컷신 중 HUD와 조준점을 숨기고 전투·투사체를 정지하며, 부드러운 카메라 이동과 타자식 대사 표시를 적용
- 최종 보스 전투 직전 주인공과 같은 외형의 `잔영-00`을 실제 맵에 배치하고, 정체성·새봄·두 사람의 이름을 다루는 7단계 대화로 확장
- 첫 입력은 남은 대사를 즉시 표시하고, 다음 입력에서 다음 대사로 넘어가도록 컷신 조작을 개선

- 스테이지 5 `원형 보관소 · 거울의 뿌리`와 7개 전용 구역, 이중 경로, 교차 레이저, 추가 봉쇄실 구현

- 잔영-00 전용 패턴: 거울 발도, 역상 산탄, 이중 도약 추격, 잔상 반격, 14방향 기억 폭주

## GitHub에 올리기

이 폴더를 저장소 루트로 사용하세요. GitHub에서 빈 저장소를 만든 뒤 아래 명령을 실행하면 됩니다.

```powershell
git init
git add .
git commit -m "Initial release"
git branch -M main
git remote add origin https://github.com/사용자명/저장소명.git
git push -u origin main
```

저장소의 `Settings → Pages → Build and deployment → Source`를 `GitHub Actions`로 선택하면 `main` 브랜치에 푸시할 때 게임이 자동 배포됩니다. 배포 주소는 Actions의 `Deploy GitHub Pages` 작업에서 확인할 수 있습니다.

로컬 실행과 문법 검사는 다음 명령으로 할 수 있습니다.

```powershell
npm.cmd start
npm.cmd test
```

## 조작

- `A`, `D`: 이동
- `Space`: 점프 / 이중 점프 / 벽차기
- 마우스 이동: 조준
- 마우스 왼쪽: 캐릭터가 바라보는 방향으로 발도술
- 마우스 오른쪽: 샷건. 공중에서 아래로 발사하면 반동으로 상승
- `F` 또는 `C`: 키보드 샷건 보조 입력
- `J` 또는 `X`: 키보드 발도 보조 입력
- `E`: 주변 탄환을 소거하는 버스트
- `R`: 마지막 체크포인트 복귀
- `Esc`: 일시정지

## 전투 순환

발도술을 적중시키거나 적 탄환을 베면 샷건 강화 게이지가 충전됩니다. 3칸이 가득 찬 상태에서 발사하면 강화 샷건이 나가며 방패를 관통하고 넓은 범위에 큰 피해를 줍니다.

샷건은 두 발을 순차 재장전합니다. 공중에서 아래쪽을 조준해 발사하면 반동으로 다시 상승하고, 수평으로 발사하면 반대 방향으로 빠르게 밀려납니다. 이중 점프를 사용한 뒤 공중 발도를 맞히면 추가 점프가 돌아옵니다.
